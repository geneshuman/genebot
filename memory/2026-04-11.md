## Lessons Learned
- `sandbox: false` is INVALID — use `{ "mode": "off" }` for disabling sandbox
- `messageContent` is NOT a valid Discord intents key in OpenClaw config
- `allow: true` at guild level is NOT a valid key — to allow all channels, remove the `channels` sub-key entirely
- `/reset` sent via `sessions_send` goes into chat as a message, not a slash command — Carlos riffs on it in-character instead of resetting
- `idleMinutes` is the simplest way to prevent old messages polluting chatbot context — no cron needed
- WS `sessions.reset` RPC requires `operator.admin` scope; scope passing via shared-secret WS auth is complex
- HTTP session reset endpoint worked for manual session clearing
